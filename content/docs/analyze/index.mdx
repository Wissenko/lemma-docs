Analysis in Lemma starts with source evidence. Use Responses to inspect individual sessions, transcripts to verify what was said, and Research Agent, Doc, Slides, or exports to synthesize patterns across participants.

## How To Review The Evidence

<Steps>
<Step>

### Step 1: Review sessions

Use Responses to scan participants, status, summaries, transcript links, and answers to each prompt.

</Step>
<Step>

### Step 2: Verify transcripts

Open transcripts when you need to check exact wording, context, or whether a generated claim is grounded in source evidence.

</Step>
<Step>

### Step 3: Ask the Research Agent

Use Chat to ask targeted questions across the study instead of reading every session manually first.

</Step>
<Step>

### Step 4: Package the findings

Use **Doc** for a written research artifact and **Slides** when the findings need to be presented.

</Step>
<Step>

### Step 5: Export when needed

Use export when the output needs to leave the Lemma workspace for a meeting, archive, or downstream workflow.

</Step>
</Steps>

## What Good Analysis Looks Like

Good analysis:

- separates repeated patterns from one-off anecdotes
- keeps exact quotes connected to transcripts
- preserves participant context instead of flattening everything into averages
- makes the sample size and audience clear
- turns findings into a decision, recommendation, or next question

## Example Study

The audited example, "How knowledge workers adopt AI tools", contains completed sessions, participant summaries, transcripts, a study chat surface, Doc output, and Slides output. Use it as a model for what a complete Lemma workflow should produce.

## FAQs & Common Issues

### Should I trust the generated summary immediately?

Use it as a starting point. Check transcripts or session summaries before turning a generated claim into a decision.

### When should I use Research Agent instead of reading sessions?

Use Research Agent when you have a targeted question across many responses. Read individual sessions when you need exact wording, context, or a participant-level story.

### When should I use Doc versus Slides?

Use Doc for a written artifact with more detail. Use Slides when the findings need to be presented or shared in a meeting.
