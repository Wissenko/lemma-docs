Properties are custom attributes stored on contacts.

They appear automatically in the segment builder, so they are useful for targeting and organizing audiences.

## When To Use Properties

Use properties for stable contact attributes such as:

- role
- company size
- plan tier
- region
- customer stage
- beta member

Avoid creating properties for one-off notes that will not be reused.

## Property Types

The audited property form supported:

- string
- number
- date
- boolean
- enum
- text_long

<HelpCallout type="tip" title="Design properties for future studies">
Create properties that will help you invite the right audience again, not just describe the current study.
</HelpCallout>

## FAQs & Common Issues

### Should I create a property for every spreadsheet column?

No. Keep properties focused on fields that help with targeting, segmentation, or analysis context.

### What is the difference between a property and a topic?

Properties are attributes you store on contacts. Topics are subscription categories participants can opt in or out of.

### What should I read next?

Use [segments](/contacts/segments) to turn properties into reusable audiences.
