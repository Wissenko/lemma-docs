This guide shows the shortest path from a blank starting point to a draft study you can review.

<Outcome title="Outcome">
You will create one study, choose text or voice, choose follow-up depth, and know where to review results.
</Outcome>

## Before You Start

You do not need polished questions. Bring a rough goal, the audience you want to learn from, and what you want to understand.

<HelpCallout type="tip" title="Use the smart builder when the questions are not clear yet">
Describe what you want to learn in plain language. Lemma can turn that into study context, starter questions, and follow-up guidance.
</HelpCallout>

## How To Create Your First Study

<Steps>
<Step>

### Step 1: Open New Study

Use **New Study** from the sidebar.

<ProductScreenshot
  src="/screenshots/docs-hq/start-quickstart-new-study.png"
  alt="The Lemma Create Study screen."
  caption="The New Study screen is the starting point for creating a study that can collect responses."
/>

</Step>
<Step>

### Step 2: Choose text or voice

Pick **Text conversation** or **Voice conversation**.

Use voice when spoken detail matters. Use text when a written path is better for the participant.

</Step>
<Step>

### Step 3: Choose follow-up depth

Pick **Quick pulse**, **Standard interview**, or **Deep dive**.

Use more depth when you need Lemma to ask more follow-up questions.

</Step>
<Step>

### Step 4: Pick a starting point

You can start from:

- Smart Builder, where you describe what you want to learn
- Upload Discussion Guide, if you already have a guide
- a template, such as Customer Discovery, Customer Satisfaction, Product/Service Feedback, or Employee Experience

</Step>
<Step>

### Step 5: Review the draft

Review the generated background, goals, tone, key terms, welcome message, closing message, and questions.

</Step>
<Step>

### Step 6: Test before using it

Use **Test Interview** or **Review** before using the study with real participants.

</Step>
<Step>

### Step 7: Know where results appear

After sessions are completed, open **Responses** for the evidence table, **Transcript** for the source wording, **Insights** for synthesis, and **Research** to ask questions across the study data.

</Step>
</Steps>

## FAQs & Common Issues

### What should I type into Smart Builder?

Type the audience, what you want to learn, and any constraints. A rough brain dump is enough.

### When should I use a template?

Use a template when your study matches a common workflow already available in the UI.

### Should I edit the questions?

Yes. Smart Builder gives you a draft. Review the questions, follow-up settings, and instructions before using the study.
