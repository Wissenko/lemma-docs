Segments are filtered groups of contacts.

Use them when you want to target a reusable audience instead of manually selecting participants each time.

## When To Use Segments

Segments help you send studies to:

- customers in a plan tier
- users in a region
- contacts with a role or function
- beta testers
- people subscribed to a topic
- contacts matching custom properties

## Before You Create A Segment

Define useful [properties](/contacts/properties) first. Properties appear automatically in the segment builder.

<HelpCallout type="tip" title="Name segments by intent">
Use names like "Churned customers - last 90 days" or "Beta users - admin role" so teammates understand when to use the audience.
</HelpCallout>

## What Good Looks Like

A good segment is:

- reusable across more than one study
- specific enough to improve answer quality
- based on stable contact data
- easy for a teammate to understand

## FAQs & Common Issues

### Should every study use a segment?

No. Use a segment when the audience will be reused or when targeting matters. For a small manual test, a direct link may be enough.

### Why is my segment too broad?

Add properties that describe the participant context you care about, such as role, lifecycle stage, plan tier, region, or customer type.

### What should I read next?

Use [topics](/contacts/topics) when participants should be able to opt in or out of study categories.
