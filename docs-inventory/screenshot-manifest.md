# Screenshot Manifest

Screenshots should come from a stable demo workspace with clean sample data.

Use neutral demo content, not real private customer data unless Wissem explicitly approves the workspace as mock/demo data. Prefer screenshots that show the main user task clearly over screenshots that show every UI detail.

## Screenshot Quality Rules

- Use high-resolution desktop captures.
- Use demo data with specific but non-sensitive examples.
- Prefer one clear action per screenshot.
- Crop only when the crop helps the user focus.
- Keep enough surrounding UI to orient the user.
- Do not use screenshots to imply unavailable features.
- Keep one screenshot per docs page unless a deliberate exception is approved.
